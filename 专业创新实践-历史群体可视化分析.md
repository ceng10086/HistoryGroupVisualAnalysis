# 专业创新实践-历史群体可视化分析(以下信息来自ppt)

## 一、 核心选题信息（基于第73页）
*   **选题方向**：历史群体可视分析（属于“可视化VIS”与“数字人文”交叉领域）。
*   **指定/推荐数据集**：**中國歷代人物傳記資料庫 (CBDB)**
*   **数据来源网址**：https://projects.iq.harvard.edu/chinesecbdb/home

## 二、 案例系统功能展示（根据第73页系统截图界面提取）
PPT中展示了一个针对明代江南文人（如沈周、文徵明、祝允明、唐寅等）的历史人物群体可视分析系统界面，系统包含以下核心可视化模块，可作为项目设计的参考：
1.  **中心人物设置（查询入口）**：允许用户输入或添加特定的核心历史人物进行关系检索。
2.  **动态社会网络（核心视图）**：通过节点和连线（点线图）展示人物之间错综复杂的社会关系网络。
3.  **人物身份分布**：通过统计图表（如散点聚类或气泡图）展示群体中“文人”、“官员”等不同社会身份的分布情况。
4.  **人物地理分布**：结合地图展示历史人物的活动轨迹或籍贯分布。
5.  **人物年表故事**：以时间轴（Timeline）的形式，标注人物在特定年份发生的重要历史事件。
6.  **人物详情信息**：右侧面板提供选中人物的详细文字资料，包含基本信息（字号、籍贯、定位、别称）、社会关系（如归属“吴门三大家”等）以及详细的人物简介（提取自史书或传记）。

## 三、 可视化（VIS）方向通用要素（基于第66页）
作为可视化类别的选题，PPT明确指出此类项目需包含以下四个关键要素：
1.  **领域数据**（如本项目中的CBDB历史人物数据）
2.  **专家需求**（即历史学家或人文领域研究者希望通过数据发现什么）
3.  **专业系统**（开发出的可视化软件或网页前端系统）
4.  **用户调研**（验证系统是否满足了目标用户群体的需求）
